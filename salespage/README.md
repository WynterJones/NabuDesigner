# NabuDesigner — sales page

The marketing/landing page that gives away the whole LP-Templates kit as a **free
ZIP download** (no signup, no checkout). Midnight design-observatory theme — deep
indigo night sky, iris-violet & aurora-teal glow, warm star-gold, and a **wise owl
mascot** (Nabu). The centerpiece is a **live showcase of all ten included templates**
(real headless-Chrome screenshots framed in browser chrome) — **click any one to open
it live in a new tab**.

Built with the same pipeline it sells (brand → mockup → fal art/video → motion → QA),
so it doubles as a reference build.

```bash
python3 -m http.server 8771        # from repo root
# open http://localhost:8771/salespage/index.html
```

## What stands out
- **Template showcase** — a featured trio + two auto-scrolling marquees of all 10
  templates, each captured live and framed in a browser window. Hover to pause; **click
  any template to open it live in a new tab** (links to `../templates/<slug>/`). This is
  the core proof: "every screen was generated by this pipeline."
- **The art pipeline band** — four cards showing the actual asset types the kit makes
  (cinematic stills, headline graphics, transparent cut-outs on a checker, looping
  video) with the fal engine used for each.
- **Cinematic owl hero** — a looping night-observatory video (ping-pong seamless),
  a drifting starfield canvas, the `Pages Worth Shipping` headline graphic, and the
  hybrid image CTA.
- **Hybrid image CTA** — one art-directed button: a CSS pill + the generated owl
  app-icon emblem + a shine sweep + a gold "FREE" chip. Reused in hero / download / final.
- **Owl host widget** — the mascot FAB opens a YouTube tour modal.
- Four-stage process timeline, feature grid, count-up stats, what's-inside checklist,
  an offer card with an `INSTANT DOWNLOAD` seal + rotating glow, FAQ accordion,
  starfield final CTA.

## The free download (the ZIP, served by GitHub Pages)

The kit is a **free** download — no checkout, no signup. The ZIP
(`salespage/nabudesigner-kit.zip`, ~44 MB) lives next to the page and is served by
**GitHub Pages** (GitHub's CDN), so it costs you no bandwidth. `DOWNLOAD_URL` in
`assets/script.js` is the same-origin path `"nabudesigner-kit.zip"`, and the `download`
attribute names the saved file. On the live site it resolves to
`https://wynterjones.github.io/NabuDesigner/salespage/nabudesigner-kit.zip`.

### Refresh the download whenever the kit changes
Rebuild the ZIP from the repo root, then commit & push — Pages redeploys it:
```bash
rm -f salespage/nabudesigner-kit.zip
zip -rq salespage/nabudesigner-kit.zip . \
  -x 'salespage/*' \
  -x 'index.html' -x '.nojekyll' \
  -x '.env' -x '.env.*' -x '*.key' \
  -x '.git/*' \
  -x 'raw/*' -x '*/raw/*' \
  -x 'outputs/*' -x '.farmwork/*' \
  -x '.DS_Store' -x '*/.DS_Store' -x '*.log' \
  -x 'node_modules/*' -x '*/__pycache__/*' -x '*.pyc'
git add salespage/nabudesigner-kit.zip && git commit -m "Refresh kit download" && git push
```
The ZIP includes `CLAUDE.md`, `PROMPT.md`, `DESIGN_GUIDELINES.md`, `README.md`,
`skills/`, `tools/`, and all of `templates/` — **not** `salespage/`, the Pages-only
root files (`index.html`/`.nojekyll`), or `.env`.

> Want to keep the 44 MB out of git instead? Upload the same ZIP to a **GitHub Release**
> and set `DOWNLOAD_URL` to
> `https://github.com/WynterJones/NabuDesigner/releases/latest/download/nabudesigner-kit.zip`
> (the script already handles the cross-origin case). GitHub serves Release assets as
> attachments, so the button still downloads directly.

**Template preview links** point at `../templates/<slug>/index.html`, so the
`templates/` folder must be served alongside `salespage/` (it is, when you serve the
repo root). If you deploy the sales page on its own, either deploy `templates/` next to
it or repoint those links.

## Re-capturing the template screenshots
The showcase images live at `assets/img/tpl-<slug>.webp`. To refresh them after a
template changes, serve the repo and re-capture with headless Chrome:
```bash
CHROME="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"
"$CHROME" --headless=new --hide-scrollbars --force-device-scale-factor=2 \
  --window-size=1440,900 --virtual-time-budget=6000 \
  --screenshot=shot.png "http://localhost:8771/templates/<slug>/index.html"
python3 tools/imgkit.py webp shot.png salespage/assets/img/tpl-<slug>.webp --max 1240
```

## Notes / assumptions
- Copy, stats and the guarantee are marketing for the real kit; the modal uses a
  placeholder YouTube ID — swap in your tour video (`data-src` on `#player`).
- All art was generated with fal (`tools/fal.py`), matted with fal `bg`, and
  post-processed with `imgkit` / `vidkit`. The template thumbnails are live captures
  of the repo's own templates.
